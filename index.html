SPA(Single Page Application)으로 구성한 웹 페이지입니다.

  • 구현 서비스 : 자기소개 서비스 CRUD 4요소(생성, 읽기, 수정, 삭제), 학과소개 서비스 CRUD 4요소(생성, 읽기, 수정, 삭제),

    •                  게시판 서비스 - 댓글기능 포함 CRUD 4요소(생성, 읽기, 수정, 삭제), 로그인 서비스 - 브라우저 세션추가 CRUD 4요소(생성, 읽기, 수정, 삭제)

  • Json-server는 REST API를 구축해주는 라이브러리

    • - 별도의 서버가 없는 환경에서 프론트엔드 작업을 진행하고자 할 때, 최소한의 실무적인 환경을 제공 받을 수 있는 API-Server 중 하나이다

    - Json-server는 Node 기반으로 구동되는 , API-Server 이다

    - 해당 프로젝트에서는 Fetch API를 활용하여 AJAX를 사용하였습니다.

    - fetch('서버주소') 의 의미는 웹 브라우저에게 이 서버주소로 요청해줘와 같은 뜻이다.

    - fetch의 뒤로 .then이 붙으면 요청이 끝나고 이 할일을 해줘 라는 뜻이다.

  • Fetch의 CRUD METHOD

    • POST : POST를 통해 해당 URI를 요청하면 리소스를 생성한다.

    GET : GET를 통해 해당 리소스를 조회합니다.

    PUT : PUT를 통해 해당 리소스를 수정

    DELETE : DELETE를 통해 리소스를 삭제
